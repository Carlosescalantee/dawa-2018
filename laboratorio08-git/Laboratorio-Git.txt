CONCLUSIONES:

• Existen dos formas de adjuntar el contenido de nuestro proyecto, con nuestro repositorio
  una es sincronizando el contenido del proyecto, con el repositorio y la otra es clonando nuestro repositorio,
  mediante la URL este, esta segunda opcion es mucho mas simple.
• Es necesario siempre hacer pull en un proyecto donde varios usuarios suben informacion, en caso no se hiciera un git pull, y agregamos un nuevo commit, 
  entonces se perderia la informacion que ya se tenga en el repositorio en la nube, y sincronizaria a lo que otro usuario tenga, algo que suele suceder en un 
  equipo de trabajo. 
• Podemos clonar nuestro repositorio en una carpeta en especifico, basta con indicarle al final de comando el nombre de la carpeta,
  y esta sera la carpeta contenedora del repositorio.
• A traves de comandos en Git agregamos, y creamos commits y subimos los archivos de nuestro proyecto a un repositorio en la nube.
• git pull nos ayuda a sincronizar el contenido de nuestro repositorio en caso de que algun usuario-colaborador hubiera hecho cambios en este,
  de forma que el contenido que se tenga en la carpeta contenedora del proyecto, va a actualizarce igual que los archivos del repositorio en la nube.
